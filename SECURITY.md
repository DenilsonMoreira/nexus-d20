# Segurança

Relate vulnerabilidades de forma privada ao responsável pelo repositório. Não abra issue pública com segredos, tokens, dados pessoais ou passos de exploração contra ambientes reais.

Prioridades:

- isolamento entre campanhas;
- proteção de notas privadas;
- autorização de ações do mestre;
- uploads e URLs assinadas;
- autenticação e sessões;
- prevenção de enumeração de IDs;
- auditoria imutável.
